# CL-001 Source Intake Contract

## Purpose

This file defines the evidence burden that was used for the four retired CL-001
packet files. It is not evidence, does not accept any source-domain claim, and
does not score any gate. Current CL-001 Interval Sweep work requires a new
source-intake contract after the frame is declared.

These retired packet fields remain open. They may not be populated for the
active CL-001 Interval Sweep.

## Boundary Rules

- Treat the WI-069 founding corpus as origin history, not evidence.
- Preserve source-domain sovereignty; this repo may cite domain evidence but
  does not own the source domain's truth or verdicts.
- Keep target and null source intake symmetric enough that nulls cannot be
  made to fail by definitional exclusion.
- Record losses, imports, boundary conditions, and open fields before scoring
  agency, escape, feedback, or absorber gates.
- Leave a packet field open when the evidence burden has not been met.
- Do not convert unlike quantities into a shared scalar unless the packet names
  a typed conversion map and its failure conditions.

## Evidence Lanes

| Evidence lane | Required before packet population | Applies to |
|---|---|---|
| Mechanism and transition boundary | Source must support the operative process and boundary/interface claimed in `T` and `B`. | All packets |
| Regime and class definition | Source must distinguish `R0`, `K0`, `R1`, and `K1` without relying on hindsight labels. | All packets |
| Typed measurement family | Source must name the admissible measurements for `M0` and `M1` and identify non-comparable quantities. | All packets |
| Loss and import accounting | Source or construction note must identify dissipation, uncertainty, external subsidies, hidden schemas, or missing imports for `L` and `I`. | All packets |
| Agency surface and action space | Source must support or explicitly fail the proposed `P`, `A`, and `Omega` entries without treating observer interest as agency. | All packets |
| Class-relative no-go or constraint | Source or construction note must state the constrained class and assumptions before `N` or `X` can be populated. | All packets |
| Feedback and recurrence | Source must support downstream effects on future regimes, interfaces, or agency before `F` can move from open to populated. | All packets |
| Absorber and null pressure | Source or construction note must name the strongest ordinary explanation tested in `Z`. | All packets |
| Falsifiers and open fields | Packet must keep falsification conditions and unresolved evidence gaps visible in `V`. | All packets |
