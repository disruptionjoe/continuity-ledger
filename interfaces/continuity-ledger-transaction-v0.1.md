# Continuity Ledger Transaction Packet v0.1

Status: proposed founding interface.

## Purpose

Represent a regime-crossing transaction without collapsing unlike quantities
into one scalar and without calling every causal chain emergence.

## Packet

```text
CLTP = (R0, K0, S0, M0, T, B, R1, K1, S1, M1,
        C, L, I, P, A, Omega, N, X, F, Z, V)
```

## Required Fields

| Field | Meaning | Required challenge |
|---|---|---|
| `R0`, `R1` | Source and target regimes | Why are these distinct regimes rather than descriptions? |
| `K0`, `K1` | Model/admissibility classes | Which assumptions changed? |
| `S0`, `S1` | Material or operational substrates | What physically or institutionally bears the process? |
| `M0`, `M1` | Typed measurement/accounting families | Which quantities are comparable, and which are not? |
| `T` | Transduction mechanism | What operative mechanism crosses the boundary? |
| `B` | Boundary/interface | What controls admission, conversion, or recognition? |
| `C` | Continuity witness | Is this equality, provenance, constraint, entitlement, or causal dependence? |
| `L` | Loss ledger | What dissipates, degrades, becomes uncertain, or leaves scope? |
| `I` | Import ledger | What oracle, subsidy, schema, energy, legitimacy, or external capacity enters? |
| `P` | Agency surface | What stable-enough regularity supports adaptive action? |
| `A`, `Omega` | Agent class and action space | What actor/process is created or recruited, and what can it now do? |
| `N` | No-go/constraint ledger | Which class and assumptions were binding? |
| `X` | Escape witness | What constructive class change enables a formerly unavailable action? |
| `F` | Recursive feedback | How does action alter future regimes, interfaces, or agency? |
| `Z` | Strongest absorber/null | What ordinary theory could explain the case without this framework? |
| `V` | Evidence/provenance | What primary sources, grades, and falsifiers support each field? |

## Gate Verdicts

Each packet receives one verdict per charter gate: `PASS`, `FAIL`, `BLOCKED`,
or `NOT_APPLICABLE`, plus a reason and evidence reference. The packet-level
verdict is one of:

- `SHARED_STRUCTURE`
- `NARROWER_STRUCTURE`
- `KILL`

No packet may pass by filling a field with a metaphor or by naming a hidden
import after the outcome.
