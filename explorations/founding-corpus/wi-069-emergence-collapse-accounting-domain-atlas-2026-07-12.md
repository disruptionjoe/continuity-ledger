Status: staged deep-research orchestration artifact

## 1. Objective

This run executes the first pre-repo research move recommended by the preflight report: discover a large, divergent selection of systems where collapse is a live structural risk, and test whether the proposed accounting structure can carry those cases without reducing them to metaphor.

The question is not "where does the word collapse appear?" The question is:

> Across many domains, what relation is lost, fixed, absorbed, canonicalized, exhausted, captured, or over-completed when a system stops producing meaningful new possibilities?

This matters because Joe's clarified target is not a TI/TAF/GU triangle. It is a cross-domain test of AI-accelerated thinking: can a research process find the same bookkeeping problem across many unrelated systems, while preserving enough technical specificity that the result is not analogy soup?

## 2. Accounting Structure v0.2

The preflight report proposed `EscapePacket`. This run sharpens it into two linked packet types.

### 2.1 CollapsePacket

```text
CollapsePacket =
  (domain,
   system_class,
   carrier,
   collapse_mode,
   absorber_or_null,
   lost_relation,
   escape_or_resilience_mechanism,
   residue_or_invariant_candidate,
   coherence_cost,
   first_test)
```

Definitions:

- `domain`: the discipline or research surface.
- `system_class`: the modeled class, with assumptions.
- `carrier`: what is supposed to carry emergence or productive capacity.
- `collapse_mode`: typed failure, not a generic negative word.
- `absorber_or_null`: the simpler model that explains the phenomenon without genuine emergence.
- `lost_relation`: the relation that collapse destroys or over-completes.
- `escape_or_resilience_mechanism`: what prevents, delays, or escapes collapse.
- `residue_or_invariant_candidate`: what may survive representation change.
- `coherence_cost`: what must be paid to avoid incoherent explosion.
- `first_test`: a concrete falsification or narrowing test.

### 2.2 Open-System Accounting Tuple

A recurring cross-domain shape appears:

```text
OSA_n = (G_n, A_n, M_n, B_n, R_n, C_n, E_n)

G_n  generation operators
A_n  admissibility / selection / evaluation rules
M_n  memory, record, provenance, retained diversity
B_n  boundary input, external contact, fresh source, or environmental openness
R_n  representation / readout / interface / observer maps
C_n  coordination, finality, consensus, legitimacy, or coherence protocol
E_n  escape obligations and no-go ledger
```

First hypothesis:

> Collapse occurs when one or more OSA fields is replaced by a fixed completion, single scalar, canonical record, closed equilibrium, captured governance path, or recursive self-generated distribution in a way that destroys the system's ability to generate, preserve, or test new admissibility.

This is not yet a theorem. It is an accounting lens.

## 3. Collapse Mode Taxonomy

The domain scan separates at least ten collapse modes.

| Code | Collapse mode | Description | Typical escape mechanism |
| --- | --- | --- | --- |
| C1 | Fixed-completion factorization | Future novelty is absorbed into a completed object plus access/projection/schedule. | Pre-action noncompletion or productive witness. |
| C2 | Readout/finality absorption | Apparent source novelty is only observer access, record stabilization, or capability readout. | Source/readout separation plus preservation maps. |
| C3 | Canonicalization / quotient collapse | Distinct possibilities are forced into one canonical record or equivalence class. | Retained residue, dissent trace, nontrivial invariant. |
| C4 | Equilibrium / absorbing-state collapse | Dynamics enter a fixed point, basin, or absorbing state with no active transitions. | Perturbation, boundary input, criticality, viability maintenance. |
| C5 | Diversity / mode collapse | Distribution loses modes, niches, classes, or behavioral diversity. | Diversity-aware selection, archives, fresh data, replay. |
| C6 | Recursive information collapse | Self-training, self-reference, or excessive copying erodes tails or true-source signal. | Fresh source data, provenance filtering, anti-recursion controls. |
| C7 | Capture collapse | Governance, incentive, or legitimacy system becomes easier to exploit as value grows. | Contestability, monitoring, bounded authority, non-capture controls. |
| C8 | Metric / objective collapse | Proxy target becomes the system's real objective, destroying the original relation. | Multi-objective review, adversarial metric tests, goal-proxy separation. |
| C9 | Error-threshold / fidelity collapse | Mutation/noise exceeds ability to preserve information or identity. | Error correction, redundancy, repair, selection balance. |
| C10 | Semantic / category collapse | Transfer is fake because terms are homonyms or class assumptions changed. | Layer 0 semantic alignment and class-relative no-go bookkeeping. |

## 5. Packet Details

### 5.1 Temporal Issuance: Completion-Factorization Collapse

Carrier:

Source-side action/admissibility generation.

Collapse:

The trace factors through fixed completed families plus access, projection, schedule, finality, naming, provenance/action, or capability maps.

Lost relation:

The relation between pre-action source state and genuinely new admissibility/action.

Escape:

E166's OSAG shape: internally generated pre-action family noncompletion rule, with imported oracle and after-fact singleton rejected.

Residue:

Class-relative local constructive source action. Not physical issuance.

First test:

Try to instantiate OSAG outside Temporal Issuance. If every attempt compiles to fixed completion, the new repo should not generalize TI.

### 5.2 Time as Finality: Readout/Capability Absorption

Carrier:

Record stability, observer shadow, and bounded-region capability `C(R,M)`.

Collapse:

Source novelty collapses into observer access, finality, region enlargement, joint-record completion, law-sector completion, transition-system completion, or description completion.

Lost relation:

The distinction between source growth and observer/capability growth.

Escape:

Predeclare task, region, observer, horizon, intervention menu, and preservation fields, then show the change is not just readout.

Residue:

Capability boundary, not emergence invariant.

First test:

Pair an OSAG-like source perturbation with a TaF `C(R,M)` model and ask whether all capability effects can be readout-absorbed.

### 5.3 GU Formalization: Class-Relative No-Go Collapse

Carrier:

Geometry/observerse structures and boundary/invariant data.

Collapse:

Bridge claim collapses when a no-go theorem applies to the same class, or when an escape is a category change, homonym, or imported target.

Lost relation:

The honest relation between richer substrate and forgetful shadow.

Escape:

Declare Layer 0 plus L1-L7; identify the exact class killed and the exact class exited.

Residue:

Boundary/index/flux datum, 2-primary blindness, hard-core no-go residue, or honest external datum.

First test:

Run every candidate domain packet through GU's Layer 0 plus L1-L7. Kill transfers that cannot specify L7 or semantic alignment.

### 5.4 Architecture of Legitimacy: Capture Collapse

Carrier:

The legitimacy operation:

```text
local contribution evidence P -> eta_P -> L(P)
```

Collapse:

Contribution records become opaque, unchallengeable, captured, coercive, or formally compliant while participants experience unfairness.

Lost relation:

The trace from local evidence to legitimate accepted record, including rejected or contested residue.

Escape:

Visibility, contestability, voice, exit with retained record, adaptive rule change, bounded authority, and non-capture monitoring.

Residue:

Accepted record plus visible contested/deferred/lost material.

First test:

Model capture profitability:

```text
attack attractive when
probability_of_success * extractable_value
  >
direct_cost + mitigation_cost + expected_sanction + reputation_loss
```

Then ask whether defense strengthens as value grows.

### 5.5 AI Epistemology: Machinery Collapse

Carrier:

Designed epistemic machinery: claim ledgers, adversarial review, absorber maps, provenance, metrics, search over search machinery.

Collapse:

AI acceleration collapses into:

- faster old methods;
- same failures in new form;
- existing formal methods;
- no transfer;
- no measurement;
- self-confirming machinery;
- training-data correlation floor.

Lost relation:

The claimed separation between thinking, motivation, incentives, verification, and execution.

Escape:

Specification-certified separability, independent transfer tests, K6 realized-output independence, matched-budget higher-order lift.

Residue:

Potential explicit evolvability of epistemic machinery, not yet established.

First test:

Use this atlas itself as a self-test: does it discover a reusable structure, or just generate a better analogy list?

### 5.6 GAN Mode Collapse

Carrier:

Generative model distribution.

Collapse:

The generator emits too narrow a set of outputs, sometimes one or few modes, instead of covering the training distribution.

Lost relation:

Generator-to-data-distribution coverage.

Escape:

Minibatch discrimination, unrolled discriminator objectives, Wasserstein distance, diversity-sensitive evaluation, better data coverage.

Residue:

Diversity/coverage cannot be reduced to single-sample realism.

Accounting lesson:

A system can optimize local acceptance while collapsing global diversity. This maps strongly onto governance and epistemic review systems where "accepted-looking" outputs can erase the possibility space.

Sources:

- Salimans et al., "Improved Techniques for Training GANs" (2016): https://arxiv.org/abs/1606.03498
- Metz et al., "Unrolled Generative Adversarial Networks" (2016): https://arxiv.org/abs/1611.02163
- Arjovsky, Chintala, Bottou, "Wasserstein GAN" (2017): https://arxiv.org/abs/1701.07875

### 5.7 Recursive Model Collapse

Carrier:

Training data distribution across model generations.

Collapse:

Models trained recursively on model-generated data forget tails of the original distribution.

Lost relation:

Contact with true source distribution.

Escape:

Fresh real data, provenance-aware filtering, limits on synthetic-data recursion, mixture thresholds.

Residue:

Boundary openness to non-self-generated data.

Accounting lesson:

Open-endedness can require continual contact with a source not generated by the system itself. This is a concrete non-metaphysical version of "source openness."

Sources:

- Shumailov et al., "AI models collapse when trained on recursively generated data" (Nature, 2024): https://www.nature.com/articles/s41586-024-07566-y
- Seddik et al., "How Bad is Training on Synthetic Data?" (2024): https://arxiv.org/abs/2404.05090

### 5.8 Catastrophic Forgetting

Carrier:

Neural network weights as shared representation of sequential tasks.

Collapse:

New learning overwrites old learning.

Lost relation:

Retention of prior task competence under adaptation.

Escape:

Replay, modularization, regularization, memory systems, task separation, continual-learning mechanisms.

Residue:

Plasticity-stability tradeoff.

Accounting lesson:

Emergence cannot mean "always change." Systems need trace-preserving change. Otherwise openness destroys memory.

Sources:

- McCloskey and Cohen, "Catastrophic Interference in Connectionist Networks" (1989): https://www.sciencedirect.com/science/chapter/bookseries/pii/S0079742108605368

### 5.9 Neural Collapse

Carrier:

Last-layer representations in trained classifiers.

Collapse:

Within-class variability collapses toward class means and a simplex ETF-like geometry during terminal training.

Lost relation:

Fine within-class distinctions in representation, but not necessarily performance.

Escape/resilience:

Not necessarily needed. This can be beneficial simplification.

Residue:

Collapse can be productive finality. This is the strongest warning against treating collapse prevention as automatically desirable.

Sources:

- Papyan, Han, Donoho, "Prevalence of Neural Collapse during the terminal phase of deep learning training" (PNAS, 2020): https://www.pnas.org/doi/10.1073/pnas.2015509117

### 5.10 Objective-Based Evolutionary Search

Carrier:

Search process over behaviors/genotypes/designs.

Collapse:

Fixed objective produces deceptive gradients and local optima.

Lost relation:

Access to stepping stones that do not look like progress toward the target.

Escape:

Novelty search, non-objective search, quality diversity, behavior archives.

Residue:

Adjacent possible is not a fixed path to objective. Diversity of stepping stones matters.

Accounting lesson:

An admissibility rule can collapse possibility by over-specifying what counts as progress.

Sources:

- Lehman and Stanley, "Abandoning objectives: evolution through the search for novelty alone" (2011): https://pubmed.ncbi.nlm.nih.gov/20868264/
- Pugh, Soros, Stanley, "Quality Diversity: A New Frontier for Evolutionary Computation" (2016): https://www.frontiersin.org/journals/robotics-and-ai/articles/10.3389/frobt.2016.00040/full

### 5.11 Digital Evolution Surprises

Carrier:

Evolutionary algorithm plus environment/objective/representation.

Collapse risk:

If treated only as objective optimization, surprising strategies get dismissed as bugs or obstacles.

Lost relation:

Evolved behavior can exploit unmodeled affordances, exposing hidden assumptions.

Escape:

Record surprising behavior as scientific data, not just failure.

Residue:

Creativity may appear as exploitation of the evaluator's blind spots.

Accounting lesson:

An emergence ledger must distinguish productive escape from reward hacking. Both look like "unexpected behavior."

Sources:

- Lehman et al., "The Surprising Creativity of Digital Evolution" (Artificial Life, 2020): https://arxiv.org/abs/1803.03453

### 5.12 Quasispecies Error Threshold

Carrier:

Genetic information in a mutating population.

Collapse:

Mutation/error exceeds threshold; information about the master sequence cannot be maintained.

Lost relation:

Fidelity between reproductive process and heritable structure.

Escape:

Error correction, redundancy, mutation-rate control, selection balance.

Residue:

Open-ended evolution requires both variation and fidelity. Too much novelty is collapse.

Accounting lesson:

This is the biological mirror of coherence-openness tradeoff.

Sources:

- "Examining the Theory of Error Catastrophe" (Journal of Virology, 2006): https://pmc.ncbi.nlm.nih.gov/articles/PMC1317512/
- "Error catastrophe and antiviral strategy" (PNAS): https://www.pnas.org/doi/10.1073/pnas.212514799

### 5.13 Ecological Catastrophic Regime Shifts

Carrier:

Ecosystem state and resilience basin.

Collapse:

Gradual pressure causes abrupt shift to another stable regime.

Lost relation:

Recoverability to prior regime under ordinary perturbations.

Escape:

Maintain resilience, monitor early warning signals, reduce pressure, preserve feedback diversity.

Residue:

Basin structure and hysteresis.

Accounting lesson:

Collapse can be classically dynamical: not completion, but loss of recoverable basin. The ledger needs a dynamical row separate from source noncompletion.

Sources:

- Scheffer et al., "Catastrophic shifts in ecosystems" (Nature, 2001): https://pubmed.ncbi.nlm.nih.gov/11595939/

### 5.14 Self-Organized Criticality

Carrier:

Spatially extended dynamical system.

Collapse edge:

The system self-organizes near a critical point rather than settling into a simple equilibrium, but it remains avalanche-prone.

Lost relation:

Not exactly lost; the system sustains scale-free activity by operating near instability.

Escape/resilience:

Distributed local dynamics maintaining criticality.

Residue:

Criticality as a non-equilibrium openness candidate, but not evidence of source emergence by itself.

Accounting lesson:

Some systems avoid fixed equilibrium by living on a boundary. The accounting structure must distinguish boundary-open dynamics from genuine source noncompletion.

Sources:

- Bak, Tang, Wiesenfeld, "Self-organized criticality" (Physical Review Letters, 1987): https://link.aps.org/doi/10.1103/PhysRevLett.59.381

### 5.15 Absorbing-State Phase Transitions

Carrier:

Active lattice/population/diffusion process.

Collapse:

Dynamics enter an absorbing inactive state with no spontaneous escape.

Lost relation:

Active transition capacity.

Escape:

External drive, noise, reproduction above threshold, conserved activity, or altered local rules.

Residue:

Active/inactive phase boundary.

Accounting lesson:

This is a clean physical model of "loss of productive capacity" without metaphysics.

Sources:

- Hinrichsen, "Non-equilibrium critical phenomena and phase transitions into absorbing states" (2000): https://arxiv.org/abs/cond-mat/0001070

### 5.16 Distributed Consensus and FLP

Carrier:

Agreement protocol among asynchronous processes.

Collapse:

Not collapse into a wrong record, but inability to guarantee termination under one crash fault.

Lost relation:

Agreement + validity + termination under the declared asynchronous fault model.

Escape:

Synchrony assumptions, randomization, failure detectors, weaker liveness, partial synchrony, or changed model class.

Residue:

Class-relative no-go exactly in the GU/TI sense: assumptions matter.

Accounting lesson:

Coherence is not free. Reconciliation can fail unless the protocol imports timing, randomness, or failure-detection structure.

Sources:

- Fischer, Lynch, Paterson, "Impossibility of Distributed Consensus with One Faulty Process" (1985): https://www.cs.princeton.edu/courses/archive/fall07/cos518/papers/flp.pdf

### 5.17 Commons and Institutional Capture

Carrier:

Shared resource plus governance rules.

Collapse:

Uncoordinated extraction, free riding, rule capture, or failure of collective-choice institutions.

Lost relation:

Participant incentives no longer preserve the shared resource or legitimate rule process.

Escape:

Ostrom-style institutional design: boundaries, local congruence, collective choice, monitoring, sanctions, conflict resolution, recognition of rulemaking rights, nested governance.

Residue:

Polycentric governance and maintained legitimacy.

Accounting lesson:

Open access without legitimate process can collapse faster as value grows. Openness needs anti-capture accounting.

Sources:

- Hardin, "The Tragedy of the Commons" (Science, 1968): https://www.science.org/doi/10.1126/science.162.3859.1243
- Ostrom, "Governing the Commons" (1990 PDF mirror): https://www.actu-environnement.com/media/pdf/ostrom_1990.pdf

### 5.18 Goodhart/Campbell Metric Collapse

Carrier:

Metric used as proxy for a goal.

Collapse:

The proxy loses validity when it becomes a target.

Lost relation:

Measure-to-construct relation.

Escape:

Adversarial metric testing, plural measures, causal modeling, qualitative review, anti-gaming controls, governance of metric updates.

Residue:

The construct cannot be replaced by one target variable without losing meaning.

Accounting lesson:

Metric collapse is C1 in social form: the system pretends the goal factors through a fixed scalar.

Sources:

- Goodhart, "Problems of Monetary Management: The UK Experience" (1975 record): https://www.econbiz.de/Record/problems-of-monetary-management-the-u-k-experience-goodhart-charles/10002525062
- Campbell, "Assessing the impact of planned social change" (1979): https://www.sciencedirect.com/science/article/pii/014971897990048X

### 5.19 Lucas Critique: Policy-Invariance Collapse

Carrier:

Econometric model relating observed variables to policy effects.

Collapse:

A statistical relation fails when policy changes the decision rules that produced the relation.

Lost relation:

Observed regularity to structural invariant.

Escape:

Model deeper decision rules/preferences/constraints, not only historical correlations.

Residue:

Policy-invariant structure, if it exists.

Accounting lesson:

An absorber can be policy-sensitive: the completion that described yesterday's behavior may collapse under intervention.

Sources:

- Lucas, "Econometric policy evaluation: A critique" (1976): https://www.sciencedirect.com/science/article/pii/S0167223176800036

### 5.20 Formal Verification: Desirable Collapse

Carrier:

Formal proof object and checker.

Collapse:

A claim collapses into a completed proof term under a fixed formal system.

Lost relation:

Ambiguity, informal interpretive latitude, and some forms of semantic richness.

Escape/resilience:

Not needed when the target is proof closure. The system wants decidable finality.

Residue:

Formal verification works precisely because the admissibility conditions are fixed and checkable.

Accounting lesson:

The new theory must not make openness a universal virtue. Some systems are valuable because they collapse correctly.

Local anchor:

- `ai-epistemology/absorbers/formal-verification.md`

## 6. Cross-Domain Pattern Findings

### Finding 1: The Same Word "Collapse" Covers Different Lost Relations

The atlas shows at least six structurally distinct lost relations:

1. Source-to-action noncompletion relation.
2. Observer-to-source separation.
3. Distribution-to-mode coverage.
4. Memory-to-adaptation preservation.
5. Governance-to-legitimacy trace.
6. Metric-to-construct validity.

These should not be merged. The accounting object can carry them, but it should not pretend they are the same quantity.

### Finding 2: Collapse Often Replaces a Relation With a Simpler Object

Across domains, collapse frequently has this shape:

```text
relation/process/context
  -> fixed object, scalar, canonical record, equilibrium, mode, or completed class
```

Examples:

- emergence -> fixed completion plus access;
- goal -> metric;
- data distribution -> few modes;
- ecological basin -> alternate stable state;
- legitimacy process -> accepted record without contestable trace;
- no-go escape -> category change without functor;
- policy behavior -> historical correlation treated as invariant.

This is the strongest reason the accounting structure may be real.

### Finding 3: Collapse Prevention Requires Both Openness and Memory

Openness alone can become noise:

- high mutation causes error catastrophe;
- too much plasticity causes catastrophic forgetting;
- open participation without filtering invites governance noise/capture;
- novelty search without archive/selection can drift.

Memory/finality alone can become closure:

- fixed completion absorbs issuance;
- canonicalization erases dissent;
- strict objective search gets trapped;
- proof closure is useful only in formal domains.

The cross-domain invariant is not "more openness." It is maintained relation among generation, admissibility, memory, boundary input, and coordination.

### Finding 4: Fresh Boundary Contact Is a Recurrent Anti-Collapse Mechanism

Examples:

- model collapse needs fresh real data;
- ecological systems need environmental feedback/resilience, not internal model recursion;
- governance needs participant contestability and external challenge rights;
- AI epistemic systems need independent scorers and K6 checks;
- TI source claims need no hidden fixed future table.

Candidate theorem shape:

> Self-generated closure without independent boundary contact tends to collapse into self-confirming distribution, captured process, or completed schedule.

This is promising but vulnerable to formal verification and closed mathematical systems, where closure is the point.

### Finding 5: Diversity Must Be Preserved as Structured Diversity

GANs, QD, ecology, commons governance, and epistemic search all show that diversity is not enough. The system needs typed, retained, and usable diversity:

- GAN diversity must cover data modes.
- QD diversity needs behavior descriptors and quality.
- governance dissent must remain visible and contestable.
- epistemic diversity must be independent enough to lower error correlation.
- ecological diversity must maintain feedback and resilience.

Unstructured plurality is not an emergence invariant.

### Finding 6: Class-Relative No-Go Thinking Transfers Well

FLP, GU, TI completion barriers, TaF capability absorbers, Goodhart, Lucas, and formal verification all benefit from the same question:

> Which class was ruled out, and what extra structure does the escape import?

That is the cleanest transferable part of the new program.

### Finding 7: A Universal Scalar Collapse Margin Is Still Not Credible

Different domains have incompatible cost measures:

- bits/probability tails for model collapse;
- basin distance/resilience for ecology;
- mutation rates for quasispecies;
- liveness assumptions for consensus;
- capture profitability for governance;
- representation covariance for GU;
- source noncompletion for TI.

The right next object is a ledger, not a scalar.

## 7. Provisional Bookkeeping Thesis

The deepest defensible thesis after this pass is:

> Open-ended systems avoid collapse when they maintain a nonfactorized accounting relation among generation, admissibility, memory, boundary contact, representation, and coordination. Collapse occurs when this relation is replaced by a fixed completion, scalar objective, canonical record, closed equilibrium, captured governance path, or self-generated distribution that no longer preserves the conditions for producing and testing new admissibility.

This is intentionally weaker than "emergence is a universal invariant."

It is stronger than "many systems collapse."

It gives a concrete next formal object:

```text
OpenSystemAccount:
  G: generation
  A: admissibility
  M: memory/provenance
  B: boundary/fresh source
  R: representation/readout
  C: coordination/finality
  E: escape/no-go ledger
```

Collapse is not one event. It is a field-specific degeneration of this account.

## 8. Where the Accounting Structure Works

Strong fit:

- TI completion barriers.
- GU no-go/escape maps.
- FLP and distributed systems.
- Goodhart/Lucas metric-policy collapse.
- AI model collapse.
- GAN mode collapse.
- catastrophic forgetting.
- commons/capture governance.

Moderate fit:

- ecological regime shifts;
- quasispecies error threshold;
- open-ended evolution/QD;
- AI epistemic evolvability.

Weak/risky fit:

- self-organized criticality, because it may be a boundary-dynamics example rather than emergence;
- neural collapse, because collapse can be productive;
- formal verification, because completion is desired;
- digital evolution surprises, because productive escape and reward hacking are hard to separate.

## 9. What Would Kill the Program

This preflight points to crisp kill conditions:

1. If `OpenSystemAccount` cannot be specified without domain-local hand waving, do not create a repo.
2. If every external example is fully absorbed by existing local theory with no reusable accounting burden, keep this as an interface artifact.
3. If "boundary contact" reduces to arbitrary external input, the theory is trivial.
4. If "admissibility growth" cannot be distinguished from changing the evaluator's preferences, the theory collapses into Goodhart/governance language.
5. If cross-domain packets fail Layer 0 semantic alignment, the atlas is a vocabulary map, not a research object.
6. If the ledger cannot generate a prediction, invalid transfer, theorem target, or executable classifier, it is not a repo-worthy program.

## 11. Recommendation Update

This domain atlas strengthens the case for a pre-repo experiment, but still does not justify creating the repo immediately.

What got stronger:

- The accounting structure transfers better than expected.
- Collapse often does look like replacement of a living relation by a fixed object, scalar, canonical record, or closed process.
- The OSA tuple gives a concrete center that is broader than TI/TAF/GU without absorbing them.
- External domains supply real divergence: AI training, ecology, distributed systems, governance, economics, open-ended evolution, and statistical physics.

What remains unearned:

- No composition theorem.
- No validated schema.
- No source-blind coding pass.
- No proof that this escapes institutional isomorphism.
- No invariant.
- No demonstration that `OpenSystemAccount` predicts a missing absorber or invalid transfer.

Updated recommendation:
