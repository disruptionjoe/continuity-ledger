## Experiment Question

Can the two live WI-069 North Star candidates classify paired examples with enough discipline to justify continued pre-repo shaping?

- Version A: recursive incentive transduction.
- Version B: class-relative measurement escape ledger.

## Method

Use two deliberately different pairs:

1. Version A pair: Bitcoin proof-of-work versus photosynthesis.
2. Version B pair: model collapse versus language archivalization.

For each pair, test whether the packet identifies:

- source measurement regime;
- target measurement regime;
- transduction or transition interface;
- lost and gained measurements;
- first-order behavior;
- second- and third-order incentive ecology;
- feedback on future measurement regimes;
- boundary import;
- residue or transported invariant;
- obstruction or absorber;
- factorization status;
- falsification or fixture path.

## Pair A: Bitcoin Proof-Of-Work Versus Photosynthesis

### Why This Pair

This is the best stress test for Version A because both examples convert an external source into a target regime that then drives further behavior. The risk is metaphor: "energy becomes value" is too loose. The packet must force separate source, interface, target quantity, behavior, feedback, and collapse fields.

### Packet

| Field | Bitcoin Proof-Of-Work | Photosynthesis |
|---|---|---|
| Source regime | Electricity, hardware, time, thermodynamic cost, hash computation | Sunlight, carbon dioxide, water, mineral context, environmental gradients |
| Interface | Proof-of-work mining, difficulty adjustment, validation, chain selection, subsidy and fee rules | Pigments, electron transport, carbon fixation, organismal metabolism, ecological coupling |
| Target regime | Ledger security, chainwork, block reward eligibility, monetary issuance, settlement finality | Chemical energy, sugars, biomass, oxygen flux, trophic support, ecological productivity |
| First-order behavior | Miners expend capital and energy to produce valid work; nodes accept the heaviest valid chain | Organisms convert light into chemical potential and grow biomass |
| Second-order ecology | Mining investment, energy-market search, fee-market expectations, custody/exchange/infrastructure layers | Food webs, plant competition, oxygen cycling, soil formation, niche creation |
| Third-order ecology | Political economy of protocol immutability, security-budget debate, jurisdictional mining competition, financialization | Planetary oxygenation, aerobic metabolism, climate/carbon cycling, evolutionary landscape shift |
| Feedback on source | Energy costs and mining revenue shape future hashpower, hardware, geography, and security budget | Biomass and ecology alter future light capture, carbon cycling, soil/water conditions, and species niches |
| Boundary import | Physical energy, chip supply, network connectivity, social monetary demand, developer/node legitimacy | Solar flux, atmospheric chemistry, water, nutrients, viable organisms, ecological continuity |
| Residue or invariant | Costly work trace under protocol rules; scarcity schedule; validation history | Stored chemical potential; biomass; oxygen/carbon flux; trophic capacity |
| Absorber | Treating energy spend as intrinsic social value; ignoring externality, centralization, or legitimacy costs | Treating sunlight availability as biological productivity; ignoring interface or downstream ecological limits |
| Factorization status | Does not reduce to raw energy. It factors only through protocol-defined costly participation and validation. | Does not reduce to raw light. It factors through biochemical interface and ecological uptake. |
| Collapse mode | Security-budget failure, hashpower centralization, externality blindness, monetary-policy legitimacy loss | Nutrient starvation, interface failure, ecological decoupling, trophic collapse |
| Fixture path | Incentive/security model separating energy input, difficulty, reward, fees, and hashpower concentration | Energy-flow/ecology model separating light input, conversion efficiency, nutrient constraints, biomass output |

### Pair A Discriminator

The Version A frame works only if "source -> interface -> incentive/capability regime -> recursive ecology" is mandatory.

The packet discriminates because raw source magnitude is not enough in either case:

- more electricity is not automatically more monetary legitimacy or security if validation, distribution, and incentive ecology fail;
- more sunlight is not automatically more ecological productivity if the biological interface or nutrient/ecological context fails.

The common structure is not "energy becomes value." The stricter structure is:

```text
source gradient
  -> constrained interface
  -> target-regime quantity
  -> behavior and ecology
  -> future source/interface/measurement conditions
```

### Pair A Verdict

Version A is viable as the audacious public hook and as a hypothesis generator. It is not yet sufficient as the repo spine unless every example is forced to name the interface, target quantity, second-order behavior, and feedback path.

## Pair B: Model Collapse Versus Language Archivalization

### Why This Pair

This is the cleanest Version B stress test because both examples distinguish record preservation from source preservation. The risk is overgeneralizing "source loss" without identifying the measurement-regime transition.

### Packet

| Field | Recursive Model Collapse | Language Archivalization |
|---|---|---|
| Source measurement regime | Independent data-generating process with tail events, provenance, and external validation | Living speech community with pragmatic use, transmission, improvisation, authority, and context |
| Target measurement regime | Generated-output corpus treated as source-equivalent training data | Archive, dictionary, grammar, recording, or standardized text treated as completed language record |
| Lost measurements | Tail mass, provenance independence, distributional diversity, source novelty | Living pragmatic competence, intergenerational transmission, community legitimacy, context-sensitive use |
| Gained measurements | Inspectable synthetic volume, controllable sampling, lower acquisition cost, benchmarkable outputs | Stable record, searchable corpus, teachable reference, external inspectability, preservation finality |
| Transition interface | Training loop, sampling policy, filtering, synthetic-data curation, evaluator | Archiving, standardization, transcription, grammar production, institutional preservation |
| Obstruction | Local plausibility does not preserve the source distribution across recursive training generations | Completed record does not preserve the source-generating practice of a language |
| No-go class | Self-consuming generative loops without sufficient independent source replenishment | Archival/completion models that equate record completeness with language vitality |
| Boundary import | Independent real data, provenance, tail audits, external evaluators | Speakers, teachers, learners, use domains, community authority, legitimate institutions |
| Residue | Distributional tail and provenance that self-output cannot regenerate | Living generativity and pragmatic competence that records cannot supply alone |
| Closure class | Terminal closure when the training loop closes only over its own outputs; productive closure when source renewal is maintained | Captured or terminal closure when archive replaces living use; productive closure when records support transmission |
| Factorization status | Collapse when future training factors through a fixed self-output distribution plus readout | Collapse when future language capacity factors through a fixed record plus retrieval |
| Falsification path | Show recursive generated-data training preserves tail diversity and external validity without independent source renewal | Show archival completeness alone sustains new competent speakers and legitimate use without living transmission |

### Pair B Discriminator

The Version B frame works because it separates "record quality" from "source vitality."

The pair is not merely analogous:

- model collapse is a measurable distribution/provenance failure;
- language archivalization is a transmission/legitimacy/pragmatics failure.

Both share a stricter transition pattern:

```text
living or independent source regime
  -> completed record/output regime
  -> readout treated as source
  -> source-generating capacity becomes invisible or degraded
```

### Pair B Verdict

Version B is the stronger research spine. It gives a cleaner falsification path, handles closure without treating all closure as collapse, and keeps the boundary import burden explicit.

## Findings

1. Version A has the best public and ideation energy, but it must be disciplined by packet fields that prevent "source becomes value" metaphor drift.
2. Version B is more publishable and more repo-spine-ready because it produces explicit lost/gained measurements, no-go classes, boundary imports, residues, and falsification paths.
3. The two versions should not be blended into one vague theory. The strongest current shape is Version B as the research spine and Version A as the audacious hook and example-harvest engine.
4. A new repo is still premature. The experiment supports continued pre-repo shaping, not immediate repository creation.
5. The next useful experiment, if Joe wants to continue, is the remaining paired-contrast set with Version B fields required and Version A fields added only where source-to-incentive recursion is present.
