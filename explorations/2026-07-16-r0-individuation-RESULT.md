## The impossibility argument, scoped

Adversary C did not claim no rule can fix `R0`. It claimed:

> No **autonomous** `R0`-individuation rule is charter-admissible,

where autonomous means the rule's inputs contain no analyst-populated CLTP
field. Its mechanism is **relocation, not elimination**: every candidate has a
non-empty presupposition set, so every candidate is a function on *packets*, not
on *cases*. It constrains the packet; it cannot ground it. The CLTP is a
relational schema — a constraint system fixes relations among relata, it does
not individuate its relata.

Two charter-internal amplifiers, both derivable from the text:

- **The charter's definition of transduction is teleological.** It reads: "A
  mechanism that changes the ... regime **through which downstream action becomes
  possible**." Identifying `T` therefore requires already identifying the
  downstream action. The dependency runs `R0` ← `T` ← `Omega`: the source regime
  is fixed by the agency you have already decided was generated.
- **The charter's invariance test cannot be applied to `R0`.** It requires an
  invariant survive redescription "relevant to its model class." The model class
  is `K0`, indexed *to* `R0`. The test presupposes what it would certify. **The
  repository contains no apparatus capable of validating an `R0` rule.**

And the Type-Distance Dilemma, evidenced on both targets: at `d`=0 there is no
transduction (Bitcoin's chainwork is typed "expected number of hashes" — a count
of the source-side activity, so `M0` and `M1` coincide and the case *is* the hash
null); at large `d` the continuity relation degenerates to causal storytelling,
the charter's own falsifier. A packet passes only at intermediate `d`, and
nothing fixes `d`. **The grain parameter is a verdict dial.**

## My preregistered outcome interpretation was refuted

The preregistration named three outcomes. Its third read:

> **NO ADMISSIBLE RULE.** ... The regime boundary is a free parameter and the
> instrument cannot discriminate. This is a `KILL` on the instrument.

The first sentence is correct. **The second does not follow, and is refuted.**

`R0` is a **gauge choice**. Gate verdicts are gauge-dependent — that much is
fatal to *absolute* verdicts. But discrimination is a **difference between two
packets**, and a difference can be computed in a fixed gauge without the gauge
being canonical. Potentials are conventional; potential differences are not.

> A declared-boundary packet **can** discriminate a target from its matched null
> — if and only if the null is evaluated at the **same** declared boundary.

So the bar's outcome list was a false trichotomy. It assumed absolute
individuation was the only route to discrimination. It is not. This is the second
of this author's predictions refuted today; the first was `C-PROBE-01`'s. Both
failed the same way: by presuming their own frame.

**Absolute individuation of `R0` is dead. Relative individuation survives, and is
stronger than what the repo has now.**

## The live defect this exposes

**The charter's null-matching criterion never mentions `R0` matching** — including
this repository's own 2026-07-16 Matched Nulls amendment. "Matched" currently
means matched-in-subject-matter, not matched-in-frame. That is a gap in the
amendment, found within hours of its ratification.

Neither current branch discriminates:

- Pick Bitcoin's `R0` = computational regime → target and null share `R0` and
  nearly share `R1` → no discrimination (null-collapse).
- Pick `R0` = participant regime → the hash null has no counterpart `R0` at all →
  the packets are not comparable, they are **incommensurable**.

## The amendment's near-miss nulls are right, for a reason it failed to state

Testnet/regtest shares mainnet's `R0`, `K0`, `S0`, `M0`, `T`, and `B` — identical
protocol, identical hashing, identical consensus — and differs *only* in the
economic `R1`. Rayleigh–Bénard shares thermal input and dissipative structure and
differs in metabolism and heredity.

**`R0`'s arbitrariness is neutralized by pairing, not by individuation.** The free
parameter becomes a *shared* parameter, and the gate *difference* becomes
evidence. This is a further, independent reason the floor nulls carry zero
weight: they cannot be populated at their target's frame at all.

**Technical correction to the amendment, on the record:** testnet3's difficulty
rules differ from mainnet's, so testnet3 is not a clean frame-match. **Regtest is
the cleaner frame-match.** The amendment named "Bitcoin testnet or a regtest
chain" without distinguishing them.

## Proposed consequences (not enacted here)

Three follow, and each requires Joe's authorization because each touches ratified
text or the experiment's design.

1. **`R0`-matching as a null-admissibility condition.** A null is admissible only
   if it can be populated at the same declared frame Φ = (`R0`, `K0`, `S0`, `M0`,
   `T`, `B`) as its target.
2. **Prefer regtest over testnet3** as near-miss Null A.
3. **The frame sweep — a test this repository does not have.** Since `R0` is a
   dial, a discriminator must be shown robust *across* the dial, not merely to
   exist at one setting. Sweep Φ over the admissible range:
   - target passes and frame-matched null fails, across the sweep →
     `SHARED_STRUCTURE` is live;
   - the pass/fail pattern **inverts** anywhere in the sweep → the discriminator
     is a frame artifact → `KILL` or `NARROWER_STRUCTURE`;
   - discrimination survives on a sub-range → `NARROWER_STRUCTURE`, with the
     surviving range as the stated domain. This is the most likely honest
     outcome, and it is a result rather than a failure.

**What would be fatal:** if no frame Φ exists at which a target passes and its
frame-matched near-miss null fails, the free parameter is not merely free — it is
doing all the work, and the charter's falsifier "matched null controls satisfy
the same gates as the target examples" fires. **On present evidence this is
untested, because CL-001 has never held a frame fixed across a target/null pair.**

## Separate finding: a provenance defect in the evidence layer

Constructor A found this while trying to build on the typed-quantity tables. It
is independent of its rule, and it is verified.

The dossiers' **"Non-comparable with" column is repo-authored, not
source-derived**, and it sits inside a section whose surrounding discipline
signals "extracted from source."

The proof is two table cells:

- `evidence/cl-001-dossiers/photosynthesis-agency-surface.md:64` — Net primary
  production, source context "Field et al. global terrestrial/oceanic production
  synthesis", non-comparable with: "Standing biomass, heat, photon count, agency,
  **Bitcoin chainwork**."
- `evidence/cl-001-dossiers/photosynthesis-mechanism.md:63` — Photon absorption,
  source context "Light-driven photosystem reactions", non-comparable with:
  "Biomass, ecological opportunity, organismal agency, **Bitcoin chainwork**."

Field et al. is *Science* 1998. Bitcoin's whitepaper is 2008. **A 1998 paper did
not name Bitcoin chainwork as a non-comparable quantity.** Neither did the
photosystem literature.

The cause is structural, in `evidence/cl-001-source-dossier-template.md`. The
Typed Quantities table is `| Quantity | Type or unit | Source context |
Non-comparable with |`. **"Source context" carries provenance for the quantity.
"Non-comparable with" has no source-attribution column and no attribution
requirement.** The template licenses an unsourced non-comparability graph inside
an evidence artifact.

The consequence: the non-comparability graph encodes **CL-001's comparison**, not
photosynthesis's type structure. It was written by an agent that already knew
which two cases the charter had paired. Any method that reads the boundary off
those cells — as Constructor A's rule did — reports the repo's own prior scoping
decisions as a discovery about the world.

Proposed, not enacted: require source attribution for each non-comparability
declaration, or rename the column to mark it repo-authored analysis rather than
extraction. Both are contract changes affecting six existing dossiers.
